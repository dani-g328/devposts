The Widening Responsibility for Front End Developers.

Danielle Gordon

In the world of web development there is two paths taken traditionally. There are the Front End Developers. They focus on browsers. Creating the webpage, making sure it is compatible with all or most browsers (ie, firefox, chrome, internet explorer). They create what the user sees.

On the other hand is the Back end developers. They work on the things users do not see. Hosting and servers are the main things they work on. Making sure the website can store data and function as the client wants.

However, in this article written by Chris Covier and published Oct 7th, 2020, he writes about how this form is changing. Chris says in this article that essentially the role of Front End Developers is growing as far as responsibility.

> Front-end development is at the intersection of art and logic. A cross of business and expression. Both left and right brain. A cocktail of design and nerdery.

New Responsibilities of Front-End Developers

  • Use Nova or similar programs to do server side work

  • Be able to create entire websites in Javascript

  • Be in a sense, the architect of entire websites

  • Compartmentalize websites into components

  • Use Javascript to track states of a website

  • Account for growing capabilities in browsers

  • Keeping up with trends and expectations from internet users.

Summery

As the world of web development grows so does the individual responsibility of front-end developers. The job market for this work is no longer only about Design and HTML skills, it requires knowledge of servers and Javascript to build more full and complex websites.

Javascript is also growing in capacity. It can do everything HTML or Css can while adding interactive elements that would require several programming languages previously. Using Javascript devs can easily compartmentalize components and manage them easier than using HTML or Css alone.
